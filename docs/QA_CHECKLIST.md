## v0.4-GEN — general CAD generation engine

These must each generate useful CAD (no "Something went wrong"):
1. Drill jig plate 120×80, 6mm thick, 6mm guide holes spaced 25mm, registration lip
   (countersunk holes no longer error).
2. a hexagonal gear with a 10mm shaft → hex profile (not a round disk).
3. a pulley with a 10mm shaft hole and 60mm OD → grooved pulley.
4. a simple bearing housing for a 20mm shaft → feature graph.
5. a rectangular block with a stepped slot and two counterbored holes.
6. a hexagonal spacer with a 6mm through hole.
7. a 90 degree pipe elbow with circular flanges.
8. a flange plate with 8 holes on a 100mm bolt circle.
9. a shaft collar with an M6 clamp screw.
10. full inline-four crankshaft prompt.
11. a 40mm cube with a 10mm hole → general planner.

Checks:
- Each design shows a Route badge (Precision template / Flexible CAD graph / SCAD
  generator) and, when defaults/repairs applied, an "Auto-repaired generation" or
  assumptions banner.
- Drawing-to-CAD with the flanged-pipe hint generates simplified CAD; a long
  dimension label or a string like "approx 90mm" no longer collapses the flow.
- SCAD fallback (when OpenSCAD installed) shows the STL-only note; STEP is never faked.
- `python -m scripts.run_generation_regression --provider openai --limit 150` (opt-in).

## v0.3.8 — generate-first

Run `APP_ENV=development LLM_PROVIDER=openai OPENAI_API_KEY=…` + frontend.

1. **Drill jig generates (no over-clarify)** — Prompt: "Drill jig plate 120mm by
   80mm, 6mm thick, with 6mm guide holes spaced 25mm and a registration lip." →
   generates a drill_jig; a "Generated with defaults" banner lists the hole
   count/lip assumptions. It must NOT ask for hole count/lip size.
2. **Bracket / enclosure / pipe-clamp** prompts all generate immediately.
3. **Crankshaft long prompt** → generates `inline_4_crankshaft` (no "outside
   supported object types"); 5 main / 4 rod journals, 6 flange bolts.
4. **Drawing (OpenAI active)** — upload the flanged-pipe drawing; if low
   confidence, add hint "flanged pipe branch with 12 holes per flange, 90mm main
   pipe, 50mm branch pipe, 10mm wall, 15mm flange" → detected
   `flanged_pipe_branch`, confidence ≥ 75%, Confirm enabled → generates.
5. **Provider errors visible** — with a bad key / no `openai` package, the drawing
   page shows the real error (key missing / package missing / model error), not
   "unknown 0%". `/api/provider-status` reflects availability.
6. **Generate with defaults** — for any clarification that's safe, the button
   appears and produces CAD; otherwise "Refine prompt".
7. **No "Something went wrong"** — errors show the backend detail.


## v0.3.7 — production interaction & real AI

Run backend with `APP_ENV=development LLM_PROVIDER=openai OPENAI_API_KEY=…` and
the frontend dev server. (For offline UI checks you may use the mock, but
Drawing-to-CAD requires `DEV_ALLOW_MOCK_DRAWING=true` and a text hint.)

1. **Provider status** — Header/Drawing page shows “OpenAI vision active”; in mock
   it shows “Mock mode — image understanding blocked”.
2. **Mock blocked in prod** — Start backend with `APP_ENV=production LLM_PROVIDER=mock`
   → startup fails with a clear error.
3. **View toolbar drives the MAIN viewer** — Generate an enclosure; click
   Top/Front/Right/Left/Iso/Fit → the main 3D viewport visibly changes each time;
   the active button is highlighted.
4. **PNG** — Click PNG → the current main viewport downloads as a PNG.
5. **Circle-to-edit selects** — Turn on Circle Edit, drag a circle over a hole →
   it shows “Selected: hole_0” and Apply enables. (Toggle “⌖ anchors” in dev to
   see projected feature points.)
6. **Circle-to-edit applies** — Type “make this hole 8 mm” → Apply → only that
   hole changes.
7. **No-feature feedback** — Circle empty space → “No editable feature found”.
8. **Drawing Views panel gone** — The right sidebar no longer shows view thumbnails.
9. **Real drawing** — With OpenAI active, upload the flanged-pipe drawing → it is
   NOT a bracket; detected type + confidence shown.
10. **Low confidence** — If confidence < 75%, “Confirm & generate” is disabled
    (“Needs clarification”); add a correction hint and re-interpret.
11. **Crankshaft long prompt** — Paste the full crankshaft prompt (>1500 chars) →
    routes through ComplexCADPlan → crankshaft generated with valid STL/STEP.
